### Phase 1: Database Models

#### [NEW] [developer.model.js](file:///Users/praveen/Code/JssConnect/Backend/src/models/developer.model.js)
We will create an isolated model just for the supreme developer to keep it separate from students/faculty.
- `email`: String (Unique)
- `password`: String (Hashed, fallback or initial login)
- `publicKey`: String (PEM formatted string representing the developer's public key)

#### [NEW] [authSession.model.js](file:///Users/praveen/Code/JssConnect/Backend/src/models/authSession.model.js)
Tracks the temporary QR code session.
- `sessionId`: String (Securely generated by `crypto`)
- `developerId`: ObjectId (Ref to Developer)
- `status`: Enum (`pending`, `verified`, `rejected`)
- `expiresAt`: Date (Sessions should expire after 5 minutes)

---

### Phase 2: Backend Node Crypto Logic

#### [NEW] [developerAuth.controller.js](file:///Users/praveen/Code/JssConnect/Backend/src/controllers/developerAuth.controller.js)
This will contain the 3 core APIs required for the QR flow:

1.  **`initiateBiometricAuth(req, res)`**:
    - Laptop sends Developer Email.
    - Uses `crypto.randomBytes(32).toString('hex')` to generate a secure `sessionId`.
    - Creates an `AuthSession` in the DB as `pending`.
    - Returns `sessionId` to the laptop to generate the QR code.

2.  **`verifyFromMobile(req, res)`**:
    - The mobile phone hits this endpoint after scanning. It sends `sessionId` and a cryptographic `signature`.
    - Backend uses `crypto.createVerify('SHA256')` to load the `signature` and verifies it against the Developer's stored `publicKey` in the DB.
    - If it matches mathematics perfectly, it updates the `AuthSession` to `verified`.

3.  **`checkAuthStatus(req, res)`**:
    - The laptop continuously long-polls this endpoint every 2 seconds.
    - Once the database says the session is `verified`, it issues an Admin/Developer JWT Token.

### Phase 3: Frontend Pages

#### [NEW] [DeveloperAuth.jsx](file:///Users/praveen/Code/JssConnect/Frontend/src/features/auth/components/DeveloperAuth.jsx)
- A secret route (e.g. `/system-override`).
- Asks for the Developer Email.
- When you click "Biometric Login", it displays the QR Code using a package like `qrcode.react`.
- Begins polling the backend `checkAuthStatus` route.

#### [NEW] [MobileVerify.jsx](file:///Users/praveen/Code/JssConnect/Frontend/src/features/auth/components/MobileVerify.jsx)
- The page the phone lands on when scanning the QR code (`/mobile-verify?session=XYZ`).
- Displays a "Verify Identity" button.
- *Note on Browser limitations*: Since real device finger-print extracting requires the `WebAuthn API`, we will use the browser's `SubtleCrypto` API to actually simulate the signing of the payload with a Private Key to show how the cryptography works under the hood so your backend `node:crypto` can verify it properly!
